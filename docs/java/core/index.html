Core Java

Below notes are based on the book OCP Oracle Certified Professional Java SE 17 Developer Study Guide.

Building Blocks​

Environment​

  1. JDK: Java Development Kit
    • javac: Convert .java (source file) into .class (bytecode)
    • java: Execute
    • jar: Package
    • javadoc: Document
  2. JRE: Java Runtime Environment
    • subset of JDK
    • could not compile
  3. IDE: Integrated Development Environment

Class Structure​

Field and Method​

  1. Method: a.k.a. function / procedure in other languages
  2. Field: a.k.a. variable
  3. Method and Field are Member of the Class.
  4. void
    • special return type
    • no value is returned
  5. Method name and parameter types are called the Method Signature.

Classes and Source Files​

  1. Class is a top level type.
  2. You can put two or more classes in a file.
  3. At most one of the top level type is allowed to be public.
  4. If you have a public top level type, it must match the filename.

main() method​

  1. main() method is for execution of the program.
    public static void main(String[] args)
    public static void main(String args[])
    public static void main(String... args)
    final public static void main(String... args)
    • public is an access modifier.
    • static is used as no object needs to be created to run the program.
    • ... is called varargs variable argument lists.
  2. Compile and run the program with commands below:
    javac Test.java
    java Test
  3. File extention must be .java.
  4. The result of compilation is .class file, i.e. bytecode.
  5. Shortcut for running single file source code.
    java Test.java
    • Skip explicit compilation step.
    • This feature is called launching single-file source code programs.
    • This feature is designed for when your program is one file.

Package and Import​

  1. Package is the logical grouping for classes.
  2. Package name is hierarchical, separated by period ( . ).
    package org.irushu;
  3. Directory structure is related to the package name.
  4. Import statement tells which package to look in for classes.
    import org.irushu.MyClass;
  5. Use wildcard to import all the classes of a package.
    import org.irushu.*;
  6. Wildcard can help shorten import list.
  7. Importing a lot of classes will not slow down execution time.
  8. Below are limitations for import statements.
    • Wildcard can only match class names, but it cannot match directories or subdirectories.
    • There can only be 1 wildcard, and it must be placed at the end.
    • Import statement can only be used to match class names. It cannot match methods or fields.
  9. Naming conflicts
    • Class names don't have to be unique across all of Java.
    • The class name you imported are therefore can be found in multiple places.
    • Explicitly importing a class name will take precedence over any wildcard present.
      import java.util.Date; // This will win
      import java.sql.*;
    • In case you need a class name from 2 different packages, use fully qualified class name to declare the fields.
      java.util.Date utilDate;
      java.sql.Date sqlDate;
  10. java.lang are automatically imported.
  11. Classes in the same package are automatically imported.

Compilation, Execution and JAR files​

  1. You can compile as below.
    javac org/irushu/test/Test.java
    javac org/irushu/test/*.java
    javac *.java // This won't compile the source file in package org.irush.test
  2. You can use -d to place class files in a different directory.
    javac -d classes org/irushu/test/Test.java
  3. You can specify the classpath for required libraries to compile the program.
  4. You can specify the classpath and run the program.
    java -cp classes org.irushu.test.Test
    java -classpath classes org.irushu.test.Test
    java --class-path classes org.irushu.test.Test
  5. You can create jar file as below.
    jar -cvf test.jar .
    jar -cvf test.jar -C classes . // -C specify the location that contains the class files
    jar --create --verbose --file test.jar .

Ordering elements in a class​

  • Elements should be ordered in a class as below.
    1. Package declaration
    2. Import statement
    3. Top level type declaration (Class / Interface / Enum)
    4. Field declaration
    5. Method declaration

Creating objects​

  1. You can define constructor for creation of objects.
  2. Below is an example of constructor. Note it does not have return type.
    public class Test{
        public Test(){
            // do something
        }
    }
  3. If you don't provide a constructor, compiler will supply a do nothing default constructor for you.
  4. You can create object with the new keyword.
    Test test = new Test();
  5. One can read and write instance variables directly from the caller. 
    public class Test{
        String helloWorld = "Hello World";
        public static void main(String... args){
            Test test = new Test();
            test.helloWorld = "Happy Valley";
        }
    }
  6. However, one should honour encapsulation and protect instance variables with access modifier.
  7. One can define Instance initializer as below.
    public class Test{
       {
            System.out.println("Test");
       }
    }
  8. Order of execution
    • Fields and Instance Initializer run in the order they appeared in the file.
    • Constructor
  9. Order matters and you can't refer to a variable before it has been defined.

Primitive Type and Literals​

  1. There are 8 primitive types.
    • byte (8), short (16), int (32), long (64)
      • Signed
      • Each numeric type is twice the size as the smaller similar type.
    • float (32), double (64)
      • Signed
      • Each decimal type is twice the size as the smaller similar type.
    • char (8)
      • Unsigned
      • short and char values can be casted to one another as their underlying data size is the same.
    • boolean
      • bit size depends on JVM implementation
  2. When a number, decimal, String or character is present in the code, they are called literals.
  3. By default java interprets numeric value as int.
  4. To specify the numeric literal as long, add l or L at the end of it.
  5. By default java interprets decimal value as double.
  6. To specify the decimal literal as float, add f or F at the end of it.
    float a = 2.0; // Cannot compile
    float b = 2.0f;
  7. Although float values can be declared with an f suffix, they are not printed with an f suffix.
    float b = 2.0f;
    System.out.println(b); // It will print 0.0
  8. One can specify a number by changing base. By default it is using decimal number system.
    int a = 017; // Octal (0-7)
    int b = 0xFF; // Hexadecimal (0-9 a-f A-F)
    int c = 0B101; // Binary (0-1)
    System.out.println(a); // Print 15
    System.out.println(b); // Print 255
    System.out.println(c); // Print 5
  9. You can put underscores in numbers. 
    int value = 1_000_000;
  10. You cannot put underscore at the beginning or end of a literal, or right before or after a decimal point.
    double a = _999.00; // CANNOT COMPILE
    double b = 999.00_; // CANNOT COMPILE
    double c = 999._00; // CANNOT COMPILE
    double d = 999_.00; // CANNOT COMPILE

Reference Type​

  1. A reference type refers to an object.
  2. A reference “points” to an object by storing the memory address where the object is located.
  3. A reference is like a pointer.
  4. A reference can be assigned to another object.
  5. A reference can be assigned to new object.

Primitive Type vs Reference Type​

  1. Primitive types have lowercase type names.
  2. Reference types have uppercase type names.
  3. Reference types can be used to call methods.
  4. Primitive types do not have methods declared on them.
  5. If primitive types are unknown, assign null to their wrapper class.

Wrapper Class​

  1. valueOf to convert to wrapper class.
    Integer a = Integer.valueOf("456"); // Wrapper
    int b = Integer.parseInt("789"); // Primitive
  2. Number classes, Boolean and Character Wrapper classes have userful helper methods:
    • byteValue(), shortValue(), intValue(), longValue()
    • floatValue(), doubleValue()
    • booleanValue()
    • charValue()
  3. Helper classes do their best to convert values but can result in a loss of precision.

Text Block​

  1. """ as Start text block and End text block.
  2. Incidental whitespace
  3. Essential whitespace
  4. Escape characters

Declaring variables​

  1. Identifiers must begin with letter, a currency symbol or undercore symbol _ .
  2. Currency symbol includes dollar ($), yuan (¥), euro (€).
  3. A single underscore is not allowed.
  4. Cannot use reserve word.
  5. Cannot use literal values like true, false or null.

Case convention​

  1. Method or Field names are declared as lowercase camel case.
  2. Constants or enum values are declared as uppercase snake case.
  3. Class and Interface names are declared as uppercase camel case.

Declaring multiple variables​

String str1, str2, str3="Foo";
int int1, float float1; // There can only be one type of declaration in a statement.
String str4, String str5; // You cannot repeat the same declaration in a statement, even they are the same.
double double1; double double2; // Semicolon separates statements, so there is no violation.
boolean boolean1; boolean2; Invalid declaration for boolean 2 as type is omitted.
  1. You can declare many variables in the same declaration as long as they are all of the same type.
  2. You cannot repeat the same declaration.
  3. There can only be one declaration, no other declaration can be made.

Initializing variables​

  1. Local variable is a variable defined within a constructor, method, or initializer block.
  2. final modifier is equivalent to declaring constants.
  3. final modifier can apply to local variable.
  4. For final variable of primitive type, you cannot change its value once it is initiailized.
  5. For final variable of reference type, you cannot change the reference, but you can modify the content the reference points to.
  6. Local variables do not have default value.
  7. Local variables must be initialized before use.
  8. For local variables that have never be used, compiler doesn't care if they are not initialized.
  9. Compiler is clever enough to determine whether the local variable has been used or not.
  10. If you pass uninitialized parameters to constructor or method, the program will fail to compile.
  11. Instance variable is a field defined within a specific instance of object.
  12. Class variable is defined on the class level and shared among all instances of the class.
  13. Both instance and class variables are given a default value once declared. You don't need to initialize them.
  14. Initialization of instance and class variables sticks to below rules.
    • null for an object
    • 0 for numeric types
    • 0.0 for decimal types
    • false for boolean
    • '\u000' for char
  15. Java does not support setting default method parameter values.
        public void method(int hello = 3){
        }

Using var for local variable​

  1. You can use keyword var to declare local variables under certain circumstances.
  2. This feature is called local variable type inference.
  3. var is only used for local variables.
  4. var will instruct the compiler to determine the type.
  5. In javascript, var can take on any type.
  6. But in java, compiler will determine the specific type for var. The type cannot be changed during runtime.
  7. var variable cannot be initialized to null during declaration. Compiler cannot infer its type.
  8. If the underlying type is a reference type, var variable can be reassigned with null value after declaration.
  9. var cannot be used in multiple variable assignment.
  10. For var, compiler will look only at the line of declaration. variable must be assigned values on the lines where they are defined.
    void methodA(){
        var a = "String A";
        var b
            = 5;
        var c; // var variable is not assigned with values on the same line they are declared.
        c = 1;
        var d = null; // compiler cannot infer the type
        var e = "String B";
        e = null;
        var f = 2, g = 2; // var cannot be used in multiple variable assignment
    }
  11. var cannot be used on method parameters.
     void methodB(var a, var b){
     
     }
  12. var is not a reserved word and allowed to be used as an identifier.
    void methodC(){
        var var = "hello"; // This will compile
    }

Variable scope​

  1. A set of braces means a new code block.
  2. Each code block has its own scope.
  3. Code block can contain another code block.
  4. Inner block can reference variables of outer block, but not vice versa.
  5. Scope for different type of varibles
    • Local variable: In scope from declaration to the end of the block.
    • Method parameters: In scope for the duration of the method.
    • Instance variable: In scope from declaration until the object is eligible for gc.
    • Class variable: In scope from declaration to end of the program.

Garbage collection​

  1. For details, refers to JVM section.
  2. System.gc() is not guranteed to do anything. JVM can ignore it.

Operators​

  1. An operator is a special symbol or operation that can be applied to operands.
  2. An operand is the variable / literal / value the operator being applied to.
  3. unary, binary and tenary operator take 1, 2 and 3 operand(s) respectively.
  4. Evaluation can be left-to-right or right-to-left.
  5. Some operators require the operand of specific type.

Order of operator precedence​

  1. Operator should follow below orders

    OperatorSymbolsEvaluation
    Post-unary operatorsexpression++, expression--LR
    Pre-unary operators++expression, --expressionLR
    Other unary operators-, !, ~, +, (type)Right-to-left
    Cast (Type)referenceRight-to-left
    Multiplication/division/modulus*, /, %LR
    Addition/subtraction+, -LR
    Shift operators<<, >>, >>>LR
    Relational operators<, >, <=, >=, instanceofLR
    Equal to/not equal to==, !=LR

    Logical exclusive OR^LR
    Logical inclusive OR|LR

Unary Operators​

  1. Requires exactly one operand to function.

    OperatorExamplesDescription
    Logical complement!aInverts a boolean's logical value
    Bitwise complement~bInverts all 0s and 1s in a number
    Plus+cIndicates a number is positive
    Negation or minus-dIndicates a literal number is negative or negates an expression
    Increment++e f++Increments a value by 1
    Decrement--f h--Decrements a value by 1

  2. Bitwise complement operator ( ~ ) flips all 0s and 1s in a number.

  3. Bitwise complement operator can apply only on byte, short, int, long, char.

  4. You can use the formula bitwise complement = -1 * number value -1 to calculate the result for bitwise complement operation.

    int a = 7;
    System.out.println(~a); // Print -8. Because -1 * 7 - 1

  5. Negation operator ( - ) reverses the sign of numeric expression.

  6. Negation operator can apply only on numeric expression.

  7. Increment and decrement operators ( ++ -- ) can apply only on numeric variable.

  8. The order on how Increment and decrement operators attached to the variable can change the behavior.

    • ++x / --y: increase / decrease by 1 and return the new value.
    • x++ / y--: increase / decrease by 1 but return the original value.
    int a = 1;
    System.out.println(++a); // print 2
    System.out.println(a++); // print 2
    System.out.println(a); // print 3

Binary Arithemtic Operators​

  1. Addition: a + b
  2. Subtraction: a - b
  3. Multiplication: a * b
  4. Division: a / b
  5. Modulus: a % b
  6. Arithmetic operators can apply on all primitives, except boolean.
  7. Addition operators ( + ) can apply on String for concatenation.
  8. You can change the precedence by wrapping with parentheses.
  9. Parentheses should be applied in a valid and balanced manner.
    • A new right parentheses match with a previous left parentheses
    • Equal number of left and right parentheses
  10. For integer values, division results in the floor value (value without anything after the decimal point) of the nearest integer that fulfills the operation.
    System.out.println(13 / 4); // Prints 3

Numeric Promotion​

  1. Numeric Promotion rules
    • If two values have different data types, promote one of the values to the larger of the two data types.
      int a = 1;
      long b = 2;
      var c = a + b; // a will be promoted to long. c will be long.
    • If one is integral and the other is floating-point, promote the integral value to the floating-point value's data type.
      int a = 1;
      float b = 2.1f;
      var c = a + b; // a will be promoted to float. c will be float.
    • Smaller data types, namely, byte, short, and char, are first promoted to int, even if neither of the operands is int. 
      byte a = 1;
      short b = 2;
      var c = a + b; // a and b will be promoted to int. c will be int.
      • Unary operators are excluded from this rule.
    • Resulting value will have the same data type as its promoted operands.
  2. Be aware of the data type of variables, intermediate values, and resulting values

Assignment Operator​

  1. Binary operator that assigns variable on the left side with the result of the equation on the right side.
  2. Promotion rule and Casting will occur during assignment.
  3. Automatically promote from smaller to larger data type.
    short a = 10;
    byte b = 20;
    int c = a + b;
    long d = a + b;
    float e = a + b;
    double f = a + b;
    System.out.println(c); // Print 30
    System.out.println(d); // Print 30
    System.out.println(e); // Print 30.0
    System.out.println(f); // Print 30.0
  4. Compilation error occurs if you try to promote from larger to smaller data type without casting.
  5. Casting is a unary operation.
  6. Compiler automatically casts smaller types to larger ones.
  7. Casting is required when converting to smaller data type.
  8. Casting is performed by placing the data type, enclosed in parentheses, to the left of the value you want to cast.
    short a = (short) 10;
    float b = 10;
    float c = (float) 10.0;
    float d = 10.0f;
    float e = 10.0;
  9. Casting can also be applied to object and reference.
  10. No conversion is performed for casting of object / reference.
  11. Casting an object only change the reference of the object, not the object itself.
  12. Overflow is when a number is so large that it will no longer fit within the data type, so the system “wraps around” to the lowest negative value and counts up from there.
    int a = (byte)( Byte.MAX_VALUE + 1 );
    System.out.println(a); // Print -128
  13. Underflow is analogy to overflow when the number is too low to fit in the data type.
    int a = (byte)( Byte.MIN_VALUE - 1 );
    System.out.println(a); // Print 127
  14. Casting can appear anywhere in an expression, not just an assignment.
  15. Compiler doesn't require casting when working with literals that fit into data types.
    byte a = 1;
    byte b = (byte) a * 2; // Because a is involved in the multiplication, third promotion rule applys, and a is casted to int during multiplication.
    byte c = 10 * 2; // Doesn't require casting
    byte d = 100 * 200; // Overflow occurs and compilers report error
    int e = 2 * 5.0;
    float f = 2 * 5.0;
    float g = 2 * 5.0f;
    double h = 2 * 5.0;
  16. When working with literals / value, the compiler has enough information for the programmer's intent.
  17. When working with variables, there is ambiguity about how to proceed, and the compilers will throw error.
  18. Result of assignment for the expression is equal to the value of the assignment.
    int a = 5;
    int b = ( a = 10 ); // First 10 is assigned to a. Then the whole expression a = 10 returns 10.
    System.out.println(a); // Print 10
    System.out.println(b); // Print 10

Compound Assignment Operators​

  1. Addition assignment: a += 10
  2. Subtraction assignment: b -= 10
  3. Multiplication assignment: c *= 10
  4. Division assignment: d /= 10
  5. Compound assignment is a glorified form of simple assignment. It performs the following:
    • built-in operation that applies the left and right side of statement.
    • assigns the resulting value on the left side variable.
    • compiler will automatically perform casting during assignment.
        int a = 10;
        long b = 100;
        a = a * b; // Compilation error as long type is assigned to int type
        a = (int) a * b; // Compilation error as casting applied to a only, and then promotes to long again during multiplication
        a = (int) (a * b); // Implicit casting from long to int
        a *= b; // Explicit casting from long to int

Equality Operators​

  1. Equality == and Inequality !=.
  2. For object type, we need to distinguish if
    • 2 objects are the same. (Using equality operator)
    • 2 objects are equivalent. (Using equals method)
  3. For object comparison, equality operator is applied to reference, not to the object that reference point to.
  4. For primitive type, there is no such distinction.
  5. Equality Operators must apply on the same type.
    int a = 1;
    float b = 2.0f;
    boolean c = true;
    String d = "true";
    int a1 = 1;
    float b1 = 3.0f;
    boolean c1 = true;
    System.out.println( a == b ); // Promotion rule to promote a to float
    System.out.println( b == c ); // Compilation error due to comparison between mixed type
    System.out.println( c == d ); // Compilation error due to comparison between mixed type
    System.out.println( a == a1 );
    System.out.println( b == b1 );
    System.out.println( c == c1 );
  6. Comparing null with null will return true in java.
    System.out.println( null == null ); // Print true

Relational Operators​

  1. >, >=, <, <=, instanceof
  2. instanceof tests whether the target object is a member of a particular class or interface at runtime.
  3. If the compiler can determine that a variable cannot possibly be cast to a specific class, it reports an error.
    Integer x = 1;
    if(x instanceof String){
        // Do something
    }
  4. instanceof on a null literal / reference will always return false.
    String a = null;
    String b = "Hello World";
    System.out.println( null instanceof Object ); // Print false
    System.out.println( a instanceof String ); // Print false
    System.out.println( b instanceof Object ); // Print true
    System.out.println( b instanceof String ); // Print true
  5. null cannot be used on the right side of instanceof.
    String a = null;
    System.out.println( a instanceof null );

Logical Operator​

  1. Logical AND &: true when both operands are true.
  2. Logical OR |: true when at least one of operands is true.
  3. Logical XOR ^: true when operands are different to each other.

Conditional Operator​

  1. Conditional AND &&: true when both operands are true.
  2. Conditional OR ||: true when at least one of operands is true.
  3. Short-circuit
    • For &&, if left side is false, right side will not be evaluated.
    • For ||, if left side is true, right side will not be evaluated.
  4. Avoid null pointer exception.
    String a = null;
    if( a != null && a.equals("Hello World")){ // Since a is null, if will short circuit immediately, and avoid NullPointerException on the right side
    }
    if( a != null & a.equals("Hello World")){ // Throw NullPointerException at runtime
    }

Ternary Operator​

  1. (boolean expression) ? ( expression that returned if boolean is true) : ( expression that returned if boolean is false)
  2. Second and third expressions in ternary operations don't need to be the same data type.
    int a = 5;
    long b = 10;
    int c = (int)((true) ? b : a);
    System.out.println(c); // Print 10
  3. Second and third expressions must make sense for the data type. Compiler can detect wrong data type.
    int a = 5;
    int c = (int)((false) ? "Hello" : a);
  4. Only one of the expression on the right will be evaluated at runtime. Beware of unperformed side effect.

Control Flow Statement​

  1. Control Flow Statement break up the flow of execution by
    • Decision making
    • Looping
    • Branching
  2. Target can be a single statement or block of statements.
  3. Using block is often preferred.

if statement​

Pattern matching (Java 14)​

  1. Java 16 introduces pattern matching for if statement using instanceof.
    if statement
    if(baobao instanceof Dog){
        Dog dog = (Dog) baobao;
        dog.bark();
    }
    if statement (With pattern matching)
    if(baobao instanceof Dog dog ){
        dog.bark();
    }
  2. From the above example, dog is the pattern variable.
  3. Avoid potential ClassCastException because type casting is performed only if the instanceof statement is true.
  4. Bad practice to reassign the pattern variable.
    if(baobao instanceof Dog dog ){
        dog = new SmallDog(); // Bad practice to reassign the pattern variable.
    }
  5. Prevent reassignment by final modifier.
    if(baobao instanceof final Dog dog ){
        // Do something
    }
  6. Can include && statement to filter.
    if(baobao instanceof final Dog dog 
        && dog.getAge() > 10 ){
        // Do something
    }
  7. Pattern variable must be a strict subtype. It cannot be the same type.
    Integer a = 1000;
    if(a instanceof Integer b){
        // Do something
    }
  8. Flow scoping means the variable is in scope only when the compiler can definitely determine its type.
  9. Even if the variable is not inside the if statement, it can still be deemed in scope by the compiler.
    void test(Number number) { 
        if (!(number instanceof Integer data))
            return;
        System.out.println(data.intValue());
    }
  10. The System.out.println statement from the previous example is not within if statement. However the compiler determines it is in scope if the instanceof operator returns true.
  11. We can conclude pattern matching is quite different from other scoping that determined scope by a pair of braces. The scope of pattern matching is determined by compiler.

switch statement​

  1. If no such case is found, the default will be called.
    int a = 999;
    switch(a){
        case 1: 
            System.out.println("1");
            break;
        case 2: 
            System.out.println("2");
            break;    
        default: 
            System.out.println("0");
            break;
    }
    Result
    0
  2. A break statement ends the switch statement immediately.
    int a = 3;
    switch(a){
        case 1: 
            System.out.println("1");
            break;
        case 2: 
            System.out.println("2");
            break;    
        case 3: 
            System.out.println("3");
            break;
        default: 
            System.out.println("0");
            break;
    }
    Result
    3
  3. Without break statement, it will match the first case statement, and executes all of the branches in the order it is found!
    int a = 3;
    switch(a){
        case 1: System.out.println("1");
        case 2: System.out.println("2");    
        case 3: System.out.println("3");
        default: System.out.println("0");  
        case 5: System.out.println("5");  
        case 4: System.out.println("4");  
    }
    Result
    3
    0
    5
    4
  4. Starting with Java 14, case value can be combined.
    Prior to Java 14
    int a = 2;
    switch(a){
        case 1: case 2: 
            System.out.println(" 1 or 2 ");
            break;
        case 3: 
            System.out.println(" 3 ");
            break;
        default: 
            System.out.println(" Default ");
            break;
    }
    Starting with Java 14
    int a = 2;
    switch(a){
        case 1, 2: 
            System.out.println(" 1 or 2 ");
            break;
        case 3: 
            System.out.println(" 3 ");
            break;
        default: 
            System.out.println(" Default ");
            break;
    }
  5. switch supports the following data type
    • byte, short, int
    • char, String
    • enum values
  6. boolean, long, float, double are not supported because of their range of values. Either too narrow or wide.
  7. case must be compile time constants. If the case value is not evaluated until runtime, it cannot be compiled.
  8. case support only
    • literals
    • enum constants
    • final constants

switch expression (Java 14)​

  1. Starting with Java 14, switch expression is supported.
    byte a = 2;
    byte b = 4;
    byte c = switch (a) {
        case 1 -> 0; // expression
        case 2, 3 -> { // block
            if (b == 4) {
                yield 10;
            } else {
                yield 20;
            }
        }
        default -> 99;
    };
    System.out.println(c);
    Result
    10
  2. It is a compact form of switch statement.
  3. It supports both expression and block.
  4. yield is provided to distinguish it from return.
  5. break statement is not required. Only one branch will be executed.
  6. Each case or default expression requires a semicolon as well as the assignment itself.
  7. It is allowed that switch expression doesn't return value.
    byte a = 3;
    switch(a){
        case 1 -> System.out.println("1");
        case 2 -> System.out.println("2");
        case 3 -> System.out.println("3");
        default -> System.out.println("default"); // Default case is optional in this case
    }
  8. If there is return value, all of the branches must either return value (for expression) or yield a value (for block).
  9. If there is return value, all possible case values have to be covered. You can either
    • Provide a default branch
    • Or cover all possible values. Tough quite impossible unless it is a enum types.
  10. Return data type must be consistent.
